Implement this repository as a portfolio-grade MVP called "Neural-Native Software: A Zero-API Paradigm."

The project must demonstrate a zero-generation action-routing pipeline:

Text input -> tokenizer -> frozen Hugging Face causal LM forward pass -> PyTorch forward hook captures pre-lm_head activation -> lightweight linear probe maps vector to action label -> confidence/OOD gate -> typed Python state machine executes the action.

Critical constraints:
- Do not use model.generate() in the core inference path.
- Do not ask the model to emit JSON, SQL, tool calls, shell commands, or action names.
- Do not parse user text with regexes or keyword matching to decide actions.
- Keep the LLM frozen; train only the linear probe/projection layer.
- Keep the toy application sandboxed and deterministic.
- Preserve the ABSTAIN class and OOD rejection gates.

Implementation phases:

1. Build a pure-Python TaskFlowKernel with CREATE_TASK, PROMOTE_TASK, COMPLETE_ACTIVE, ARCHIVE_COMPLETED, TOGGLE_FOCUS_MODE, and ABSTAIN. All state mutation must go through typed Action enums.

2. Implement Hugging Face model loading with optional 4-bit quantization. Implement PreLMHeadActivationTap using register_forward_hook on model.lm_head. Implement extract_vectors() that returns one final-token vector per user request.

3. Generate a synthetic intent dataset with roughly 50 variations per executable action and 100 ABSTAIN examples. Train a scikit-learn LogisticRegression probe over extracted activations and serialize it with joblib.

4. Implement an end-to-end CLI: user text -> activation vector -> probe -> OOD gate -> VectorActionPort -> TaskFlowKernel. Show route confidence, margin, OOD score, and state transition.

5. Add tests for app state transitions, vector port behavior, hook behavior, probe routing, and training. The fast test suite must pass without downloading a large model.

6. Polish the repository for CV/GitHub: README architecture diagram, quickstart, metrics table, safety boundary, limitations, and research extensions including layer sweep, activation patching, and latent-space geometry plots.

Acceptance checks:
- make dataset creates data/intent_dataset.csv.
- make test passes.
- train_probe.py can serialize artifacts/probe.joblib from an .npz feature file.
- No production action routing code calls model.generate().
- No text parser determines the action.
